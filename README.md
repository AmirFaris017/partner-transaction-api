
Partner Transaction API

Overview:
This project is a RESTful API built using .NET 9 that processes partner transaction requests.
It includes validation, signature verification, discount calculation, logging, and Docker support.

Features:
-Partner authentication (Base64 password)
-SHA-256 signature validation
-Tiered discount calculation with additional rules
-FluentValidation for request validation
-Structured logging using log4net
-Dockerized for easy deployment
-Unit tests using xUnit

Project Structure:
PartnerTransactionAPI/
├── Controllers/ 
│   └── TransactionController.cs        # API endpoint, orchestrates validation & logic
├── Models/
│   ├── TransactionRequest.cs           # Incoming request shape
│   ├── ItemDetail.cs                   # Item object within request
│   ├── TransactionResponse.cs          # Outgoing response shape with factory methods
│   └── PartnerConfig.cs                # Partner credentials model
├── Services/
│   ├── SignatureService.cs             # SHA-256 → Base64 signature generation & validation
│   └── DiscountService.cs              # Tiered discount calculation with prime & cap logic
├── Validators/
│   ├── TransactionRequestValidator.cs  # FluentValidation rules for request
│   └── ItemDetailValidator.cs          # FluentValidation rules for each item
├── logs/
│   └── transaction.log                 # Auto-generated by log4net at runtime
├── Dockerfile                          # Multi-stage Docker build
├── .dockerignore                       # Files excluded from Docker image
├── log4net.config                      # log4net rolling file configuration
├── appsettings.json                    # Partner credentials & app configuration
└── Program.cs                          # App startup, DI registration, log4net setup

PartnerTransactionAPI.Tests/
├── DiscountServiceTests.cs             # 8 unit tests covering all discount tiers & edge cases
├── SignatureServiceTests.cs            # 4 unit tests covering sig generation & tamper detection
└── TransactionRequestValidatorTests.cs # 6 unit tests covering all validation rules

How to Run:
Option 1 — Docker (Recommended)
Prerequisites: Docker Desktop installed and running.

1. Build the image
docker build -t partner-transaction-api .

2. Run the container
docker run -d -p 5000:5000 --name trx-api partner-transaction-api

3. Verify it's running 
docker ps

API is now available at: http://localhost:5000

1. Stop the container
docker stop trx-api

2. Remove the container
docker rm trx-api

Option 2 — Local .NET
Prerequisites: .NET 9 SDK installed.

1. Restore packages
dotnet restore

2. Run the app
dotnet run

Logging

All requests and responses are logged to `logs/transaction.log` using log4net.
-Passwords are **masked** (`*****`) in all log entries
-Failed validations are logged as `[WARN]`
-Successful transactions are logged as `[INFO]`
-Log file rolls over daily

Viewing logs when running locally
The log file is created at:
bin\Debug\net9.0\logs\transaction.log

Viewing logs when running via Docker:
View the log file inside the container
docker exec trx-api cat /app/logs/transaction.log

Or follow logs live
docker exec trx-api tail -f /app/logs/transaction.log
